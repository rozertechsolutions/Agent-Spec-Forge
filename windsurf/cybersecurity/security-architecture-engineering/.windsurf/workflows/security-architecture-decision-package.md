# Security Architecture Decision Package

Use this manual workflow only after the relevant Skill has produced draft material.

1. Confirm title, reference, scope, owner, reviewer, approver, date, source set, and decision audience.
2. Summarize assets, assumptions, evidence, limitations, dependencies, status, severity, confidence, actions, and residual risk.
3. Identify human decisions and approval needs separately from recommendations.
4. Attach independent architecture assurance findings or mark review as still required.
5. State completion criteria and unavailable evidence.

Do not approve architecture, accept risk, deploy, configure live systems, operate controls, publish externally, or represent drafts as approved.
