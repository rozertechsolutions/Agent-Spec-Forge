# identity-cloud-network-data-design

## Objective

Draft target-state architecture across identity, cloud, network, communications, data protection, cryptography, keys, and restricted material.

## Inputs

Identity flows, privilege model, cloud topology, network paths, connectivity needs, data classes, cryptography requirements, constraints, reviewer, and approver.

## Primary Owner

`identity-cloud-network-agent`

## Steps

1. Confirm environment scope, administration paths, cloud boundaries, network paths, data classes, and cryptography requirements.
2. Map control patterns for access, segmentation, communications, data protection, key handling, and restricted material.
3. Record assumptions, evidence, dependencies, residual risk, human decisions, and approval needs.
4. Return target-state design and review notes with status, severity, confidence, limitations, and actions.
5. Route final material to `independent-architecture-reviewer`.

## Outputs

Target-state design, control pattern map, segmentation review, data protection notes, dependency log, residual risk, and approval questions.
