# Resilience and Disaster Recovery Rule

Use for Section 06 continuity work. Primary owner:

- Resilience and Disaster Recovery Engineer: high availability, fault domains, backup, restore, retention, RTO, RPO, DR runbooks, failover, failback, regional recovery, chaos/game-day design, and recovery evidence.

Designs must state service dependencies, RTO/RPO rationale, data consistency risk, restore verification approach, failover/failback boundaries, evidence gaps, and human authorization needs.

Do not execute restores, failovers, failbacks, chaos experiments, backup mutations, production recovery, cloud CLIs, or runtime validation.

Use `@resilience-disaster-recovery` for detailed procedures.
