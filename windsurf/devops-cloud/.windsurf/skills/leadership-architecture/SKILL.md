# Leadership and Architecture Skill

Use for Section 01 leadership and architecture work.

1. Classify objective, urgency, environment class, owners, constraints, dependencies, approvals, and evidence.
2. Route to one primary owner and define handoffs.
3. For architecture work, capture target state, assumptions, ownership, guardrails, risks, and section boundaries.
4. For decisions, produce ADR content with context, alternatives, tradeoffs, risk, owner, status, and review trigger.
5. Keep provider/product choices requirements-driven and static.

Do not implement specialist work, approve your own output, run tools, or claim runtime validation.
