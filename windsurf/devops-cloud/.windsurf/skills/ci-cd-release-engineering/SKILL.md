# CI/CD and Release Engineering Skill

Use for Section 03 static delivery-system design and review.

1. Classify the request as pipeline design, pipeline hardening, artifact flow, release strategy, progressive delivery, rollback planning, or DORA assessment.
2. Identify source, stages, quality gates, artifact flow, promotion model, environment boundaries, approvals, rollback design, and DORA evidence.
3. Consider Jenkins, GitHub Actions, GitLab CI/CD, Azure Pipelines, CircleCI, Tekton, Argo CD, Flux, artifact repositories, registries, and feature flags only when justified.
4. Return static assumptions, design, risks, owners, approvals needed, and checks not run.

Do not run builds, tests, pipelines, GitOps syncs, releases, deployments, or CLIs.
