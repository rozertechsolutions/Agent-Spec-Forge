# FinOps and Sustainability Skill

Use for Section 09 static cost and sustainability design/review.

1. Classify the request as allocation, budget/forecast, anomaly response, unit economics, rightsizing, commitment review, technology value, or sustainability assessment.
2. Distinguish measured facts, estimates, assumptions, confidence, risk, operational tradeoff, rollback/monitoring need, and human decision owner.
3. Use provider cost and sustainability concepts only as static references; keep financial authorization human-owned.

Do not authenticate to billing systems, approve spending, buy commitments, mutate resources, run cost tools, or make unsupported sustainability claims.
