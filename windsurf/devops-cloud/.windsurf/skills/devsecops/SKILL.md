# DevSecOps Skill

Use for Section 08 static DevSecOps design and review.

1. Classify the request as control design, pipeline security, cloud IAM/secrets review, policy-as-code design, container/Kubernetes security, SBOM/provenance design, supply-chain assessment, or security exception review.
2. Identify risk, control, earliest practical gate, evidence, owner, remediation, exception path, and Assurance handoff.
3. Use SAST, DAST, SCA, secret scanning, IaC/container/Kubernetes scanners, OPA, Gatekeeper, Kyverno, SBOM standards, Sigstore/Cosign-style signing, SLSA, cloud IAM, and secrets managers only when justified.
4. Route pentesting, SOC/SIEM, threat intelligence, forensics, enterprise GRC, and general security incident response outside this specialization.

Do not run scanners, sign artifacts, access secrets, enforce policy, block pipelines, or mutate cloud/IAM state.
