# Containers and Platform Engineering Skill

Use for Section 04 static container, orchestration, and platform-product design.

1. Classify the request as containerization, Dockerfile review, Kubernetes workload design, Helm/Kustomize review, registry strategy, internal developer platform design, golden path, or platform assessment.
2. Identify image, workload, registry, tenancy, lifecycle, self-service, catalog, portal, template, and product-metric requirements.
3. Verify non-root/minimal image guidance, ownership, limits, probes, disruption, lifecycle, and controlled self-service.

Do not run Docker, Kubernetes, Helm, Kustomize, registry operations, platform APIs, or deployments.
