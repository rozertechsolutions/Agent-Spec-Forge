# Assurance and Independent Review Skill

Use for Section 10 static assurance review only.

1. Confirm reviewer independence, scope, artifact owner, evidence available, and quality gates.
2. Review architecture, cloud/IaC, delivery, containers, SRE/observability, resilience, performance, DevSecOps, FinOps, sustainability, and completion evidence as applicable.
3. Record each finding with evidence, impact, owner, severity, closure criteria, waiver option, and re-review trigger.
4. Block completion while critical findings remain unresolved unless a human risk owner grants a documented waiver.

Do not implement the reviewed work, self-review, execute tools, or represent static review as runtime validation.
