# ML, RAG, GenAI, Agent, and Inference Release Review

1. Confirm candidate version, task, data/model/provider lineage, inference contract, user controls, fallback, abstention, latency, cost, and operational impact.
2. Use `ml-genai-system-evaluation` for feature leakage, baselines, evaluation slices, robustness, calibration, RAG groundedness, citation correctness, prompt/tool risk, agent limits, red-team results, and release thresholds.
3. Use `ops-risk-assurance` for responsible AI, model risk, supplier risk, monitoring, drift, provider change, incident path, rollback, and retirement.
4. Require independent evaluation and independent final assurance before a release-readiness claim.
5. Return `PASS`, `FAIL`, or `BLOCKED`.
