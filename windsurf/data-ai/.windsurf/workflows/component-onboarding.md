# Source, Dataset, Model, Provider, and Component Onboarding

1. Identify component owner, supplier, purpose, data classes, allowed use, license, version, retention, access policy, and downstream consumers.
2. Use `data-contract-dataset-readiness` for source, contract, schema, semantics, dataset, label, synthetic-data, lineage, and acceptance evidence.
3. Use `ml-genai-system-evaluation` for external model, embedding, reranker, prompt, tool, agent, inference, and evaluation implications.
4. Use `ops-risk-assurance` for supplier/model risk, monitoring, rollback, deletion/rectification propagation, and retirement criteria.
5. Block onboarding when provenance, permitted use, risk owner, or human approval is missing.
