# junie — Marketing Professional

This package covers strategy, research, audiences, positioning, product marketing, customer journey, content, copy, SEO, social, email, paid media, CRO, analytics, localization, compliance, and independent quality review through the native entry points listed below.

## Native entry points
- `.junie/AGENTS.md`
- `.junie/skills/*/SKILL.md`

## Operating method
1. Load the branch-wide instructions before invoking a specialist procedure.
2. Validate the intake and assign one accountable owner per deliverable.
3. Use only the relevant native role or procedure; do not activate every component by default.
4. Keep external connections and write permissions user-configured and disabled until explicitly approved.
5. Complete independent compliance/security and quality review.
6. Return a decision package for an authorized human. Do not execute external or production actions.

## Safe defaults
- No credentials, tokens, private URLs, production identifiers, live account values, active MCP servers, or preselected providers/models are included.
- No publication, send, spend, campaign activation, audience upload, production deployment, migration, or deletion is authorized.
- Files outside this branch are not dependencies and must not be modified by this package.

## Package map
- Native instructions define branch-wide governance.
- Native roles divide exclusive ownership where the platform supports role files.
- Native Skills or procedures contain reusable execution contracts.
- `reference/` contains governance, handoff contracts, catalogs, and source notes.
- `workflows/` contains sequential professional processes; a platform-native workflow directory is also used when available.
- `templates/` contains substantive input, review, blocker, and approval records.
