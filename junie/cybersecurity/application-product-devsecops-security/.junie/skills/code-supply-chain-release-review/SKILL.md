# Code Supply Chain Release Review

Review minimum necessary source, configuration, manifests, lockfiles, SBOMs, build definitions, pipeline files, provenance, artifacts, approvals, and release evidence. Record confirmed issues separately from hypotheses requiring runtime validation. Do not execute code, scanners, package managers, pipelines, or release actions.
