# Defensive Security Operations, Detection, and Intelligence

This package is a static, organization-neutral cybersecurity configuration for Defensive Security Operations, Detection, and Intelligence. It uses supplied repository files and user-provided redacted evidence only. It does not execute generated content, install software, authenticate services, connect integrations, scan targets, deploy changes, publish findings, approve risk, or close issues.

## Professional Coverage

- Workflows covered: SOC operating-model review, telemetry-source onboarding design, detection design and review, supplied-alert triage, threat-hunt planning, threat-intelligence assessment, defensive malware-analysis planning, SOAR playbook design, detection-coverage assessment, SOC quality review, incident escalation.
- Native roles: soc-governance-telemetry-agent, detection-triage-hunting-agent, intelligence-malware-automation-agent, coverage-quality-assurance-agent, incident-escalation-reviewer.
- Reusable Skills or procedures: soc-telemetry-governance, detection-triage-hunting, intelligence-malware-automation, coverage-quality-assurance, incident-escalation-handoff.
- Independent review is mandatory for high-impact conclusions, closure recommendations, risk decisions, executive reporting, external-facing outputs, and any output with incomplete evidence.

## Evidence Standard

Every output distinguishes confirmed facts, probable findings, hypotheses, not reproduced items, false positives, accepted risks, insufficient evidence, and not-applicable criteria. Missing evidence is reported as unavailable and never converted into proof.
