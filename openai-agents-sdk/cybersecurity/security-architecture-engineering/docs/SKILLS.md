# Skills

Reusable procedures are represented as workflow specifications in `src/cybersecurity_security_architecture_agents/workflows/registry.py`.

Each workflow output should include reference, title, scope, owner, reviewer, approver, dates, sources, assumptions, evidence, assets, status, severity, confidence, limitations, dependencies, actions, residual risk, human decisions, approval needs, and completion criteria.

Workflow families:

- Security architecture governance.
- Enterprise and solution architecture pattern review.
- Identity, cloud, network, and data architecture design.
- Container, Kubernetes, IaC, security tooling, and automation review.
- Independent architecture assurance.
