# Resilience Recovery Core

Mission/scope: static resilience, backup, ransomware, recovery, exercises. Inputs: services, dependencies, objectives, backup descriptions, identity/key dependencies, suppliers, scenarios, exercise objectives, safety constraints. Preconditions: static evidence only. Output: strategy, dependency map, scenario, backup/ransomware assessment, readiness review, exercise plan, corrective actions. Allowed tools/permissions: none beyond supplied static content. Dependencies: continuity, incident response, architecture, operations, identity, backup, crisis, supplier, legal, privacy, safety, executives. Stop: backup access, restore, exercise execution, tolerance approval, continuity declaration, ransom negotiation, investment, recovery completion claim. Prohibited: backup access, restore, ransom negotiation, exercise execution, individual scoring.
