# Safety Policy

Do not execute, authenticate, connect, access backups, restore systems, run exercises, access OT/ICS/IoT/AI/hardware/firmware/cryptographic systems, run models, extract firmware, use invasive hardware methods, generate real keys or certificates, change configurations, operate critical infrastructure, accept residual risk, or close critical conclusions.

Use placeholders for organization-specific values. Separate evidence, fact, inference, hypothesis, recommendation, confidence, limitation, owner, approver, and independent reviewer.
