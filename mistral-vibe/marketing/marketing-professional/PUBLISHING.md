# Publish to Vibe Work

1. Review `PROJECT_INSTRUCTIONS.md` and create a dedicated Vibe Work project.
2. Create each folder under `skills/` as a Skill in Studio or Work, preserving its `SKILL.md`.
3. Keep Skills private during validation; publish to the workspace only after independent review.
4. Add only the role and reference files needed by the project.
5. Configure connectors separately, read-only first, without committing credentials.
6. Run a synthetic low-risk acceptance task before using organizational data.

No connector, workflow, publication, sending, spending, or campaign activation is enabled by this package.
