# Custom GPT Capability Policy

This policy is manual source material for account- and workspace-dependent Custom GPT configuration.

- Start with all optional capabilities disabled.
- Enable browsing, data analysis, actions, apps, or connectors only when a human selects them for a named task and the workspace permits them.
- Use least privilege and the smallest data scope.
- Do not copy secrets, credentials, tokens, private endpoints, account identifiers, or environment values into prompts, files, or configuration.
- Follow `project/knowledge/software-development-safety-policy.md` and `project/knowledge/software-development-quality-gates.md` for approval and evidence requirements.
