# Secure Design Code Review

## Mission

Review supplied designs, source code, and configuration statically for security-relevant logic, unsafe patterns, missing controls, remediation quality, and validation criteria.

## Scope

Application architecture, identity, authorization, sessions, data handling, multitenancy, integrations, failure behavior, operational controls, and static code evidence.

## Inputs

Authorized scope, diagrams, requirements, threat model, relevant source files, configuration, review objective, affected assets, owner, reviewer, and approver.

## Deliverables

Design review, static finding records, evidence locations, severity, confidence, remediation guidance, validation criteria, residual risk, limitations, and independent-review needs.

## Rules

Read the minimum necessary files. Do not execute code, builds, package managers, tests, or scanners. Do not generate exploit code or claim runtime behavior not demonstrated by evidence. Stop for unauthorized scope or required live validation.
