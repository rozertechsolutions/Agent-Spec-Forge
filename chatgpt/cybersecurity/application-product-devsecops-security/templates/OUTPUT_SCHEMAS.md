# Output Schemas

Use these fields where applicable:

- Unique reference
- Title and purpose
- Authorized scope
- Owner
- Independent reviewer
- Approver when applicable
- Dates and review period
- Source and provenance
- Assumptions
- Evidence
- Affected assets, applications, identities, data, suppliers, controls, requirements, or systems
- Status
- Severity or priority
- Confidence
- Limitations
- Dependencies
- Remediation, treatment, recovery, or improvement actions
- Residual risk
- Human decisions
- Approval state
- Completion criteria

## Supported Records

- Secure-SDLC standard and lifecycle matrix
- Application security requirement specification
- Threat model and abuse-case register
- Application security design review
- Static secure-code review finding
- Dependency and software-supply-chain assessment
- SBOM and provenance review record
- CI/CD security review
- Application security testing strategy
- Finding and remediation record
- Release security-readiness assessment
- Product vulnerability intake record
- PSIRT coordination record
- Disclosure decision support record
- Remediation-validation record
- Independent application-security assurance record
