# Responsibility Model

## Domains

1. Product Security Governance and Secure SDLC.
2. Application Security Requirements.
3. Threat Modeling and Abuse-Case Analysis.
4. Secure Design and Application Architecture Review.
5. Static Secure Code Review Guidance.
6. Software Composition, Dependency, and Supply-Chain Security.
7. CI/CD, Build, and Release Security.
8. Application Security Testing Governance.
9. Product Security Incident and Vulnerability Coordination.
10. Application Security Assurance and Independent Review.

## Human-Only Decisions

Humans retain final authority for release approval, deployment, exception approval, risk acceptance, disclosure, public communication, legal position, regulatory applicability, active testing authorization, and critical finding closure.

## Evidence Handling

Every conclusion must identify source, provenance, affected assets, assumptions, confidence, limitations, dependencies, and completion criteria. Static findings must not be represented as dynamically reproduced unless supplied evidence proves that status.
