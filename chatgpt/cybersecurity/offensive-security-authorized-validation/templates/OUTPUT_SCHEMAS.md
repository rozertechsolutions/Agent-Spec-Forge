# Output Schemas

Common fields: unique reference, title, purpose, authorized scope, owner, independent reviewer, approver, dates, review period, source, provenance, assumptions, evidence, affected assets, systems, identities, applications, data, suppliers, controls, requirements, status, severity, priority, confidence, limitations, dependencies, remediation or improvement actions, residual risk, human decisions, approval state, and completion criteria.

Structured outputs: security assessment authorization checklist; rules of engagement; penetration-test plan; adversary-emulation campaign plan; red-team campaign plan; white-team and deconfliction plan; purple-team exercise plan; social-engineering assessment plan; breach and attack simulation plan; offensive test case; offensive security finding; evidence manifest; cleanup and artifact-removal plan; authorized retest plan; remediation-validation record; offensive assessment report; independent offensive assurance record.

Every output must state prohibited actions, stop criteria, required human approvals, source limitations, sensitive-data minimization, and whether independent review is required before use.
