# Output Schemas

Common fields: unique reference, title, purpose, authorized scope, owner, independent reviewer, approver, dates, review period, source, provenance, assumptions, evidence, affected assets, systems, identities, applications, data, suppliers, controls, requirements, status, severity or priority, confidence, limitations, dependencies, remediation, treatment, recovery or improvement actions, residual risk, human decisions, approval state, and completion criteria.

Structured outputs: cyber resilience strategy; critical-service dependency map; cyber disruption scenario; backup and ransomware-resilience assessment; recovery-readiness assessment; cyber exercise plan; exercise after-action report; OT or ICS security assessment; IoT and embedded security assessment; AI system security assessment; AI agent and tool-use security assessment; hardware and firmware security assessment; cryptographic inventory and agility plan; post-quantum transition readiness assessment; critical-infrastructure resilience assessment; specialized corrective-action plan; independent resilience assurance record.

Every output must state prohibited actions, specialist-review needs, human authority gates, evidence limits, safety and continuity constraints, and whether independent review is required before use.
