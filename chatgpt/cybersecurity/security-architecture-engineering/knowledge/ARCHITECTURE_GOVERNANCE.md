# Architecture Governance Knowledge

## Mission

Security Architecture and Engineering turns approved business, security, privacy, resilience, and risk constraints into secure technical designs and reusable control patterns. It defines requirements and validation criteria; humans retain approval and production authority.

## Reusable Capability Routing

- Principles, decision records, governance gates, reference architecture lifecycle, escalation, and drift: `security-architecture-governance`.
- System context, trust boundaries, data flows, dependencies, threat coverage, failure modes, findings, and remediation roadmap: `enterprise-solution-architecture`.
- Human, service, workload, machine, and privileged identity patterns: `identity-access-architecture`.
- Cloud, platform, shared-responsibility, network segmentation, secure communications, endpoint, workspace, logging, guardrail, and handoff patterns: `cloud-platform-network-endpoint-architecture`.
- Data protection, cryptography, keys, certificates, container, Kubernetes, IaC, control integration, automation boundary, rollback, and observability patterns: `data-crypto-container-automation-architecture`.
- Independent traceability and sufficiency challenge: `independent-architecture-assurance`.

## Mandatory Workflows

1. Perform a security architecture review.
2. Create or maintain a reference security architecture.
3. Design identity and privileged-access architecture.
4. Review cloud or platform architecture.
5. Design network segmentation and secure communications.
6. Design data protection and cryptographic controls.
7. Review container, Kubernetes, or IaC architecture.
8. Create a security engineering control pattern.
9. Validate an architecture remediation.

## Evidence Integrity

Each output must identify source and provenance, authorized scope, assumptions, evidence quality, limitations, confidence, residual risk, dependencies, owner, independent reviewer, and human decisions. Missing, stale, partial, contradictory, or unverifiable evidence must be marked as such.

## Human Authority Gates

Human review is required for high-impact architecture decisions, identity and privileged-access patterns, cryptography and key custody, resilience, privacy, legal, records, procurement, business continuity, safety, production change, risk acceptance, exception approval, external communication, funding, and finding closure.
