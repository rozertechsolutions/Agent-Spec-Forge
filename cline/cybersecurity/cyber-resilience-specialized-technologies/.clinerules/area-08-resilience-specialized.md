# Area 08 Resilience Specialized Rules

Use static evidence only. Produce resilience strategy, dependency map, disruption scenario, backup/ransomware assessment, recovery-readiness review, exercise plan, OT/ICS assessment, IoT/embedded assessment, AI system assessment, hardware/firmware assessment, cryptographic agility plan, critical-infrastructure assessment, corrective-action plan, and assurance record.

Do not access backups, restore systems, execute exercises, access OT/ICS/IoT/AI/hardware/firmware/cryptographic systems, run models, extract firmware, use invasive hardware methods, generate real keys or certificates, change configurations, operate critical infrastructure, accept residual risk, or close critical conclusions.
