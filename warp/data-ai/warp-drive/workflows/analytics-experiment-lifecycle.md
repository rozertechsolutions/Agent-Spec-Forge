# Analytics Study and Experiment Lifecycle

1. Define hypothesis, decision, population, metric, data version, treatment/exposure, comparison, guardrails, and time window.
2. Use `analytics-experiment-review` to evaluate design, causal assumptions, sample size, power, missing data, segmentation, uncertainty, calibration, and interpretation.
3. Use `data-platform-analytics-quality` for metric definitions, semantic layer changes, dashboards, and reproducible reporting.
4. Require human review when results influence high-impact decisions, eligibility, access, pricing, safety, or user rights.
5. Return `PASS`, `FAIL`, or `BLOCKED`.
