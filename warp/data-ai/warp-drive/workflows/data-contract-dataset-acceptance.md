# Data Contract, Semantic Design, and Dataset Acceptance

1. Establish source owner, steward, model owner, data contract owner, consumers, and evidence owner.
2. Use `data-contract-dataset-readiness` to review schema, constraints, glossary, taxonomy, ontology, master/reference data, lineage, collection, labeling, leakage, splits, quality, versioning, and acceptance.
3. Use `data-platform-analytics-quality` for pipeline, data product, observability, freshness, completeness, reconciliation, and downstream metric impact.
4. Require independent review before accepting datasets used for analytics, ML, GenAI, RAG, or high-impact decisions.
5. Return `PASS`, `FAIL`, or `BLOCKED`.
