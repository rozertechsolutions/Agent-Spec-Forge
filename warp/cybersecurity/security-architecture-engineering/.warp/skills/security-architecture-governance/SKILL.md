# Security Architecture Governance

## When To Use

Use when the user asks for architecture governance, reference model, standards mapping, review gate, ownership, or decision record work.

## Instructions

1. Confirm scope, source period, standards, owners, reviewers, approver, and decision forum.
2. Inventory source artifacts and classify gaps.
3. Map standards, architecture principles, and controls to decisions and reusable reference patterns.
4. Identify owners, review gates, assumptions, dependencies, residual risk, and approval needs.
5. Route final material to `independent-architecture-reviewer`.

## Outputs

Governance model, reference index, standards map, decision record, approval questions, completion criteria, and unresolved limitations.
