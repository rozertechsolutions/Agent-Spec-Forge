# Containers and Platform Engineering Skill

Use this skill for section 04 static container, orchestration, and platform-product design. Do not run Docker, Kubernetes, Helm, Kustomize, registries, portals, Crossplane, or platform APIs.

## Procedure
1. Classify the request as containerization-design, dockerfile-review, kubernetes-workload-design, helm-and-kustomize-review, container-registry-strategy, internal-developer-platform-design, golden-path-design, platform-product-assessment.
2. Identify image, workload, registry, tenancy, lifecycle, self-service, catalog, template, golden-path, and platform-product requirements.
3. Verify minimal/reproducible/non-root image guidance, explicit Kubernetes ownership/limits/probes/disruption/lifecycle, policy-bounded self-service, and measurable platform outcomes.
4. Use Docker, OCI, Docker Compose, Kubernetes, Helm, Kustomize, container registries, Backstage-style service catalogs, developer portals, Crossplane, and platform APIs only when justified by requirements.
5. Return static evidence, handoffs, risks, and approvals needed.

## Stop criteria
Stop on requested execution, installation, authentication, embedded secrets, real endpoints, unsafe self-service, unsupported platform behavior, or missing human approval.
