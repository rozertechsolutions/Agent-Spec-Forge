# Authorization Assessment Planning

Mission/scope: authorization, rules, assessment planning, safety gates. Inputs: written authorization, scope, dates, owners, objectives, methods, prohibited actions, dependencies, constraints, evidence needs. Preconditions: explicit scoped human approval. Output: checklist, rules, plan, test case. Allowed tools/permissions: supplied files only, read-only. Dependencies: target owner, legal, privacy, safety, engineering, operations, business, SOC, incident response, third party. Stop: missing authorization, unclear owner, scope issue, safety uncertainty, prohibited method, active testing request. Completion: scope, exclusions, dates, methods, data handling, deconfliction, stop criteria, evidence, cleanup, approvals. Prohibited: creating authorization, scope expansion, treating exposure as permission, executing tests.
