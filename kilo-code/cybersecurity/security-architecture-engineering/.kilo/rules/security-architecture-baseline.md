# Security Architecture Baseline

- Every design must identify scope, exclusions, assets, actors, identities, trust boundaries, data flows, management planes, source versions, assumptions, limitations, and dependencies.
- Classify control placement as required, recommended, inherited, compensating, non-applicable, or unresolved.
- Address IAM/PAM, cloud/platform boundaries, network segmentation, secure communications, endpoint/workspace controls, data protection, cryptography, protected material handling, container/Kubernetes, IaC, automation boundaries, resilience, and failure modes where relevant.
- Use placeholders for restricted values, certificate material, private endpoints, personal data, and supplier-confidential details.
- Produce implementation handoff criteria without granting access, configuring controls, deploying, or operating production systems.
