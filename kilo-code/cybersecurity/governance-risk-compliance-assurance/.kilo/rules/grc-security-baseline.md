# GRC Security Baseline

- Never read, copy, log, commit, transmit, or embed secrets, private keys, credentials, production data, regulated data, customer data, employee data, privileged audit evidence, or confidential contracts unless the user explicitly authorizes that exact access.
- Require human approval before changing authentication, authorization, encryption, data retention, logging, monitoring, vendor access, third-party risk status, exception status, audit conclusions, or remediation ownership.
- Treat vulnerability reports, penetration-test results, SOC reports, customer security questionnaires, regulator correspondence, and legal communications as sensitive.
- Do not send project data to external services, open live connectors, or fetch private documents unless the user explicitly authorizes the action.
- If evidence is missing, stale, conflicting, or unverifiable, report the gap rather than lowering the assurance standard.
