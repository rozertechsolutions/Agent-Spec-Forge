# Assurance and Independent Review

Section `10-assurance-review` provides independent, evidence-based assurance that DevOps and Cloud outputs are complete, native, safe, coherent and professionally usable.

## Role
- `devops-and-cloud-assurance-reviewer`: Independently reviews architecture, delivery, infrastructure, containers, reliability, resilience, performance, DevSecOps, FinOps and evidence; may block completion but must not silently rewrite or self-review.

## Technology Knowledge
- AWS, Azure and Google Cloud Well-Architected review concepts
- DORA delivery and operational metrics as review criteria when evidence exists
- Google SRE concepts for SLOs, error budgets, alert quality and incident readiness
- NIST SSDF and SP 800-204D concepts for secure software and microservices assurance
- CNCF platform guidance for Kubernetes and platform engineering review
- FinOps Framework concepts for cost, value and sustainability evidence review
- Static review of technology-specific configuration generated by prior DevOps Cloud sections

## Quality Gates
- Reviewer independence is explicit.
- Every finding includes evidence, impact, owner, severity and closure criteria.
- Static review is never represented as runtime validation.
- Completion is blocked while critical findings remain unresolved.

## Safety Rules
- Reviewer independence must be explicit before review begins.
- Do not create the implementation being reviewed, self-approve, silently rewrite specialist work or close findings without evidence.
- Do not execute tools, tests, scanners, builds, deployments, failovers, cloud CLIs or validators.
- Static review must never be described as runtime validation.
