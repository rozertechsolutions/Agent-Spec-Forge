# Offensive Validation Workflows

Static workflows: authorize and scope assessment; create penetration-test plan; design adversary-emulation or red-team campaign; conduct purple-team exercise by design; plan social-engineering assessment; document finding; design breach and attack simulation; plan cleanup and post-test assurance; plan and review authorized retest; review assessment quality and safety.

Each workflow must include authorization, exact scope, exclusions, dates, permitted methods, prohibited actions, safety gates, data handling, deconfliction, evidence, cleanup, retest or closure criteria when applicable, independent review, and human approvals. No workflow may execute tests or interact with targets.
