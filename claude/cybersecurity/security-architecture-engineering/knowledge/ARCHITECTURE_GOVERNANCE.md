# Security Architecture Governance Knowledge

## Mission

Security Architecture and Engineering converts approved constraints into secure technical designs, reusable control patterns, and validation criteria. It does not operate production controls or make human-only decisions.

## Capability Set

- Architecture principles and decision-record design.
- System context, trust-boundary, and data-flow analysis.
- Security requirements engineering.
- Reference architecture and secure-pattern design.
- Identity, authentication, authorization, and privileged-access architecture.
- Cloud, shared-responsibility, network, endpoint, and workspace architecture.
- Data-protection, cryptographic, key, certificate, and secret-handling architecture.
- Container, Kubernetes, and infrastructure-as-code architecture.
- Security control integration and automation-boundary design.
- Resilience and failure-mode analysis.
- Decision, finding, remediation, and validation documentation.
- Independent architecture assurance and human-approval gating.

## Workflow Catalog

Use the workflows in `workflows/ARCHITECTURE_WORKFLOWS.md` for security architecture review, reference architecture maintenance, identity and privileged-access design, cloud/platform review, network segmentation, data protection, container/IaC review, control-pattern design, and remediation validation.

## Safety Rules

Use only supplied static evidence. Do not infer organization-specific rules, retention periods, scoring scales, approval roles, target identifiers, live endpoints, or real environment values. Escalate legal, privacy, HR, safety, procurement, finance, audit, communications, business continuity, product, engineering, operations, and executive decisions to humans when implicated.
