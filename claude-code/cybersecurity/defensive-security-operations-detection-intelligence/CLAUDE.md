# Claude Code Area 05 Instructions

Work only inside `claude-code/cybersecurity/defensive-security-operations-detection-intelligence/`.

Mission: design and review defensive security operations, monitoring coverage, detections, alert triage, threat hunts, threat-intelligence use, defensive malware-analysis plans, SOAR playbooks, coverage maps, metrics, and independent SOC assurance from static evidence.

Boundaries:

- Do not perform live queries, hunts, rule deployment, alert closure, account action, host isolation, indicator blocking, data deletion, containment, malware access, malware execution, authentication, external connection, dependency install, or generated-file execution.
- Do not own enterprise risk acceptance, production architecture, vulnerability remediation, declared-incident command, forensic acquisition, legal notification, or offensive campaign execution.
- Declared incidents transfer primary ownership to Incident Response, DFIR, and Recovery; SOC support remains evidence and monitoring support only.
- Preserve source provenance and separate evidence, inference, confidence, operational impact, human decision, and handoff owner.

Required outputs include SOC operating model, telemetry requirement, logging coverage map, detection spec, validation plan, lifecycle record, alert triage record, hunt plan/report, intelligence requirement/assessment, defensive malware-analysis plan, SOAR playbook design, coverage assessment, KPI/KRI definition, quality review, escalation record, and assurance record.
