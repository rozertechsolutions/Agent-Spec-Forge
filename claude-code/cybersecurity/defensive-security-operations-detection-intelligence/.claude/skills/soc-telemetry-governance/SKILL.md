# SOC Telemetry Governance

Use when establishing or reviewing SOC operating models and telemetry onboarding.

Steps:
1. Identify SOC scope, service hours, intake, severity model, escalation path, and decision authorities.
2. Define telemetry purpose, required events, normalization needs, privacy limits, retention expectations, and coverage evidence.
3. Produce a SOC operating model, telemetry requirement, and logging coverage map with assumptions and gaps.
