# independent-appsec-assurance

1. Confirm reviewer independence, artifact scope, owner, acceptance criteria, source evidence, and approval path.
2. Check traceability, evidence quality, severity, exploitability, missing attack paths, duplicated findings, assumptions, and limitations.
3. Detect unsupported closure, self-review, release approval drift, and claims that testing occurred without supplied proof.
4. Return severity-ordered findings, evidence gaps, residual risk, approval requirements, and readiness recommendation.
5. Block completion when evidence is insufficient or human authority is required.

Do not approve your own output, accept risk, close findings, or represent static review as dynamic testing.
