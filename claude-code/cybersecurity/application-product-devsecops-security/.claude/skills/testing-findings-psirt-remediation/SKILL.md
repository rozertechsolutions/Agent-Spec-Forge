# testing-findings-psirt-remediation

1. Define proportionate SAST, DAST, SCA, IaC, container, API, mobile, and manual-review strategy without execution.
2. Preserve supplied finding source, scanner, version, scope, and evidence.
3. Normalize duplicates and classify confirmed, probable, not reproduced, false positive, or needs validation.
4. Assess exploitability, impact, reachability, exposure, confidence, and remediation criteria.
5. For product vulnerabilities, confirm intake authorization, affected versions, confidentiality, validation plan, customer impact, remediation, retest, disclosure dependencies, and human approvals.
6. Validate remediation evidence statically and classify confirmed, partial, insufficient, or not verifiable.

Do not run scanners, access live targets, contact researchers, disclose vulnerabilities, or close critical findings through self-review.
