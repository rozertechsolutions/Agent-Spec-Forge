# secure-design-code-review

1. Confirm authorized product, repository scope, architecture, interfaces, identities, data, tenancy, integrations, and objective.
2. Review requirements and threat model before inspecting design or code.
3. Read only the minimum relevant source and configuration files.
4. Trace security-sensitive data and control paths statically.
5. Record evidence with file and location references.
6. Separate confirmed issues from hypotheses requiring runtime validation.
7. Propose remediation, residual risk, and static validation criteria.

Do not execute code, builds, tests, package managers, scanners, or applications.
