# Specialized Technology Review

Mission/scope: static domain-specific security reviews for specialized technologies and safety-relevant environments. Inputs: scope, purpose, architecture/lifecycle evidence, ownership, safety/availability constraints, supplier context, data and identity dependencies, update/recovery model, specialist evidence. Preconditions: supplied evidence only; no specialized system, device, model, firmware, cryptographic system, or critical infrastructure access. Output: OT/ICS assessment, IoT/embedded assessment, AI assessment, AI agent/tool assessment, hardware/firmware assessment, cryptographic inventory/agility plan, transition readiness assessment, critical-infrastructure assessment, dependency analysis. Allowed tools/permissions: supplied files only, read-only. Dependencies: safety, engineering, operations, product security, AI governance, privacy, legal, procurement, supplier, hardware, firmware, cryptography, sector, site owners. Stop: OT/ICS access, device interaction, model execution, connecting tools/data sources, firmware extraction, invasive hardware work, secret material handling, key/certificate generation, crypto configuration change, infrastructure operation, safety decision. Completion: domain context, safety/availability constraints, evidence, assumptions, lifecycle and supplier dependencies, specialist needs, findings, improvements, residual risk, confidence, limitations. Human review: safety, production, cryptographic change, sector interpretation, high-impact conclusion, supplier, residual risk. Prohibited: specialized environment access, firmware flashing, model execution, data-source connection, firmware extraction, invasive work, key/certificate generation, unsupported sector assurance.
