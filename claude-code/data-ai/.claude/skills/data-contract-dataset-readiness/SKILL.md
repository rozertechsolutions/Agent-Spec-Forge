# Data Contract Dataset Readiness

1. Confirm data owner, steward, permitted use, lawful or allowed basis, classification, retention, consent or restriction, access policy, and exception path.
2. Define producer, consumer, schema, constraints, units, freshness, completeness, allowed values, nullability, semantics, lineage, version, and change process.
3. Assess provenance, collection method, representativeness, coverage gaps, deduplication, synthetic-data controls, leakage/contamination risk, splits, and versioning.
4. For labeling, define instructions, reviewer qualification, adjudication, gold set, defect taxonomy, inter-annotator target, and acceptance threshold.
5. Return acceptance status, required fixes, evidence gaps, owners, monitoring, refresh triggers, and status `PASS`, `FAIL`, or `BLOCKED`.
