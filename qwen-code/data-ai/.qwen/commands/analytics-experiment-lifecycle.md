# Analytics Experiment Lifecycle

1. Run `analytics-experiment-review`.
2. Review metric definitions, target population, data source, assignment, sample size, power, confounders, interference, bias, uncertainty, calibration, sensitivity, guardrails, and interpretation limits.
3. Route governed metric and BI concerns to `data-platform-analytics-reviewer`.
4. Route privacy, consent, and decision-impact concerns to `data-ai-solution-governance-reviewer`.
5. Stop on unsupported causal claims, missing reproducibility, missing uncertainty, or sensitive data exposure.

Return validity findings, metric certification decision, interpretation limits, required changes, blockers, and exactly `PASS`, `FAIL`, or `BLOCKED`.
