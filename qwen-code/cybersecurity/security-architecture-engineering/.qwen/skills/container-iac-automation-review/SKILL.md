# Container IaC Automation Review

## Objective

Review orchestration, IaC, security tooling, and automation design for safe architecture patterns.

## Sequence

1. Confirm assets, orchestration boundaries, IaC modules, automation purpose, owner, reviewer, and approver.
2. Map guardrails, policy gates, image and runtime assumptions, restricted material handling, and human approval points.
3. Document status, severity, confidence, limitations, dependencies, actions, and residual risk.
4. Separate design recommendations from any production action request.
5. Request independent review before formal use.

## Outputs

Orchestration review, IaC guardrail map, automation package, residual risk notes, approval questions, and completion criteria.
