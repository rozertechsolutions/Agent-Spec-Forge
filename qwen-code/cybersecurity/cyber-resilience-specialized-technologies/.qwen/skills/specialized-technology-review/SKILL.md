# Specialized Technology Review

Mission/scope: static specialized technology reviews. Inputs: scope, purpose, evidence, ownership, safety/availability constraints, supplier context, dependencies, update/recovery model. Preconditions: supplied evidence only. Output: OT/ICS, IoT/embedded, AI, hardware/firmware, cryptographic, transition, critical-infrastructure assessments. Allowed tools/permissions: supplied files only, read-only. Dependencies: safety, engineering, operations, product security, AI governance, privacy, legal, procurement, supplier, hardware, firmware, cryptography, sector, site owners. Stop: access, device interaction, model execution, data connection, firmware extraction, invasive hardware work, secret handling, key/certificate generation, config change, safety decision. Completion: context, constraints, evidence, assumptions, dependencies, specialist needs, findings, improvements, residual risk, confidence, limitations. Prohibited: access, flashing, model execution, extraction, invasive work, key/certificate generation, unsupported assurance.
